⚙ Capa 3 — ETL (Extract · Transform · Load)

Proceso de extracción, normalización y carga estructurada. Los datos crudos se transforman en hechos y dimensiones compatibles con el esquema en copo de nieve del Data Warehouse.

ETL ACTIVO

📤 EXTRACT

  • Lee Excel / CSV cargado
  • Conecta SGA via API REST
  • Obtiene datos Turnitin
  • Extrae logs de LMS
  • Carga tablas relacionales
→

🔄 TRANSFORM

  • Normaliza nombres y IDs
  • Estandariza escalas (0–20)
  • Calcula score_riesgo final
  • Clasifica tipo de fraude
  • Genera surrogate keys
  • Hashea datos sensibles
→

📥 LOAD

  • Inserta en FactIncidente
  • Actualiza DimEstudiante
  • Carga DimDocente
  • Actualiza DimTiempoAcad
  • Registra en audit log

TRANSFORMACIONES SQL EJECUTADAS

-- C3: Normalización de score y clasificación de riesgo WITH cte_normalizado AS ( SELECT id_estudiante, nombre, apellido, facultad, ROUND(nota_final, 2) AS nota_norm, similitud_turnitin, CASE WHEN score_riesgo BETWEEN 0 AND 39 THEN 'BAJO' WHEN score_riesgo BETWEEN 40 AND 69 THEN 'MEDIO' ELSE 'ALTO' END AS nivel_riesgo FROM staging.incidentes_raw WHERE _staging_status = 'APROBADO' ) INSERT INTO dw.FactIncidenteFraude (sk_estudiante, sk_tiempo, nota_final, similitud_turnitin, nivel_riesgo, score_riesgo) SELECT e.sk_estudiante, t.sk_tiempo, n.nota_norm, n.similitud_turnitin, n.nivel_riesgo, s.score_riesgo FROM cte_normalizado n JOIN dw.DimEstudiante e ON e.id_estudiante = n.id_estudiante JOIN dw.DimTiempoAcad t ON t.semestre = n.semestre;
